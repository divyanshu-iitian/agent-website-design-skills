# Accessibility and Conversion Checks

## Decision Clarity

The page should make the next action obvious, explain what happens after clicking, and answer the user's biggest objection near the relevant CTA.

## Forms

Use labels, helpful placeholders only as examples, input types, autocomplete, error text tied to fields, and a visible success state.

## Navigation

Users should know where they are, how to return, and how to reach the primary action without reading the whole page.

## Accessibility

Use semantic headings, landmarks, accessible names, visible focus, readable contrast, keyboard support, and meaningful alt text for informative images.

## Trust

Add proof close to high-commitment actions: testimonials, screenshots, customer names, security notes, refund terms, privacy guarantees, or support expectations.
