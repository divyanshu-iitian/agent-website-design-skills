# Landing Page Decision Framework

## Build the Argument

Use sections only when they answer a real decision question:

1. **Relevance** - Is this for someone like me?
2. **Outcome** - What becomes better?
3. **Mechanism** - How does it work?
4. **Evidence** - Why should I believe it?
5. **Fit** - Does it handle my use case and constraints?
6. **Risk** - What will this cost in money, time, data, or commitment?
7. **Action** - What happens after I click?

The order can change with audience awareness. A search visitor comparing tools may need differentiation and price early; an unfamiliar-category visitor may need mechanism first.

## First Viewport

Include enough context to establish:

- recognizable category or offer;
- specific audience or use case when needed;
- differentiated outcome;
- primary next action;
- immediate visual or factual proof.

Do not require a slogan to carry all five jobs.

## Evidence Map

Match claims to appropriate evidence:

| Claim | Useful evidence |
| --- | --- |
| Faster or cheaper | method, baseline, timeframe, measured result |
| Easier | short workflow, time-to-value, onboarding example |
| Trusted | named customer, attributable quote, case study, certification |
| High quality | inspectable output, before/after, methodology |
| Secure | concrete controls, data handling, audit or policy link |
| Compatible | named integrations, requirements, tested environments |

If evidence is unavailable, narrow the claim.

## CTA Design

- Use a verb that describes the next step: `Start free`, `Book a demo`, `View examples`.
- Make the destination and commitment level predictable.
- Use secondary actions to answer uncertainty, not compete for attention.
- Provide feedback for loading, success, error, and already-completed states.

## Friction and Trust

Near high-commitment actions, disclose relevant price, trial terms, cancellation, setup time, support, privacy, eligibility, availability, or refund terms. Never use preselected consent, disguised ads, fake countdowns, or obstructive cancellation.

## Final QA

- Test every primary CTA and form path.
- Check real copy at narrow and wide widths.
- Confirm focus order, errors, success messages, and status announcements.
- Verify analytics identifiers without blocking the action when analytics fail.
- Check image sizing, font loading, and avoidable layout shift.
