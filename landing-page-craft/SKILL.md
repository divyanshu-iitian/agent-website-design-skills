# Landing Page Craft

Use this skill to build a landing page that communicates quickly, looks specific to the offer, and guides action without feeling like a generic template.

## Workflow

1. Identify the literal offer: product, service, venue, person, course, event, community, app, or tool.
2. Make the H1 the brand/product/place/person name or literal category. Put value proposition detail in supporting copy.
3. Use a first viewport that shows the actual offer or result. Use real imagery, generated bitmap imagery, product UI, or an immersive interactive scene.
4. Keep hero text out of cards. For hero pages, use image or scene backgrounds with readable text treatment.
5. Let a hint of the next section show on mobile and desktop.
6. Sequence sections around user decisions: what it is, why it matters, proof, how it works, details, pricing or next step, FAQ.
7. Use CTAs with specific verbs and destinations. Avoid repeating the same CTA mechanically.
8. Add proof appropriate to maturity: screenshots, logos, metrics, testimonials, examples, case studies, founder credibility, security notes, or social proof.
9. Make mobile scanning excellent: short sections, clear anchors, no text overlap, no huge empty vertical gaps.
10. Verify the page in browser screenshots before finishing.

## Section Patterns

- Hero: offer, concrete supporting copy, primary action, visual proof.
- Problem: name the pain in user language, not category jargon.
- Product tour: show actual capabilities with screenshots or UI states.
- Proof: use specific outcomes, recognizable examples, or trust markers.
- Pricing: state the decision clearly; hide complexity only when it is truly unavailable.
- FAQ: answer objections that would stop action.
